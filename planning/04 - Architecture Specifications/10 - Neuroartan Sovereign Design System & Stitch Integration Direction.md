# Neuroartan Sovereign Design System & Stitch Integration Direction

---

## 1. Direction

Neuroartan will establish a sovereign design system that can use Stitch by Google as an acceleration layer without making Stitch the source of authority.

Stitch may be used to generate interface references, design modes, component patterns, and visual exploration.

The final authority must remain the Neuroartan sovereign design token system.

---

## 2. Strategic Rule

Stitch is an accelerator.

It is not the design authority.

The design authority is:

`Neuroartan Design Token System`

---

## 3. Production Design Flow

The approved direction is:

`Stitch → Design Extraction → Neuroartan Tokens → CSS / SwiftUI / JS → Website / ICOS / Apps`

---

## 4. Cost-Free Requirement

The design system must remain startup-safe and cost-free wherever possible.

The preferred stack is:

- local files
- Markdown documentation
- JSON design tokens
- CSS variables
- SwiftUI token generation
- local component registry
- optional Stitch reference generation
- optional MCP bridge only if free and stable

---

## 5. Required Design Token Domains

The system must define:

- color
- typography
- spacing
- radius
- shadow
- opacity
- border
- motion
- elevation
- layout
- component states
- navigation primitives
- surface hierarchy

---

## 6. Propagation Targets

The design system must propagate to:

- Neuroartan website
- ICOS macOS app
- future iOS app
- future mobile platform
- Developer Mode
- Control Center
- dashboards
- runtime console
- institutional UI surfaces

---

## 7. Implementation Doctrine

No page-specific visual drift is allowed.

No one-off design patches are allowed.

All repeated UI primitives must derive from central tokens.

All future generated UI must be normalized into Neuroartan tokens before adoption.

---

## 8. Next Actions

1. Audit Stitch capabilities.
2. Identify export format and MCP availability.
3. Create Neuroartan design token registry.
4. Create CSS token output.
5. Create SwiftUI token output.
6. Create component primitive registry.
7. Propagate to website first.
8. Propagate to ICOS app second.
